- **⚠️ PRE-PUSH GATE — verify with a CLEAN RELEASE build, not incremental Debug (learned 2026-06-01):** `dotnet build -c Debug` (incremental) can report **green on a state CI rejects**. It happened: a commit shipped `.razor` *markup* (`_scale`/`Print`/`_error`/`RowClass`/`ResetFilters` on AP & AR) **without the backing `@code`**, the incremental Debug build passed, but CI's clean Release publish failed with `CS0103` and the deploy went red (the live site kept serving the prior deploy, so "live = 200" did NOT mean the push succeeded). The Razor source generator only re-emits a component's C# when its own file changes, so stale `obj/` artifacts mask cross-symbol breaks. **Before every push, run `rm -rf obj bin && dotnet build -c Release` (≈30–60s) and require "Build succeeded".** Treat a green incremental Debug build as non-authoritative. Also: a failed deploy is silent on the live URL — confirm the latest `gh run list` entry is `success` AND its `headSha` matches HEAD, don't infer success from a 200.
